Relación y dependencias

En OpenArchi cada componente puede (o no) tener asociados un conjunto de características (features). En ocasiones éstas características pueden anidarse, agruparse o relacionarse entre sí, específicamente unas con otras, para poder dar cabida a la solución de una necesidad o caso de uso particular

De éste modo puede verse como relacionar un feature específico de un componente a otro. Esta práctica será útil en una avaluación del flujo end2end de algún caso de uso, identificando qué features deben ser resueltos prioritariamente

Otro uso de esta estrategia sería limitar la concatenaci´no de un feature a otro, o restringirlo de acuerdo con ciertas condiciones
